On the understanding side, the task is object \textbf{counting}. On the generation side, the task is \textbf{count-conditioned image synthesis}. Both tasks share the same underlying concept of object quantity, providing a common evaluation axis for measuring whether numerical knowledge learned from one objective transfers to the other. We set up two complementary training conditions: training with only the understanding objective and evaluating generation (Und.\ $\rightarrow$ Gen.), and vice versa (Gen.\ $\rightarrow$ Und.).

In this section, we examine whether knowledge transfer between understanding and generation emerges in current open-weight UMMs. Following the categorization in Fig.~\ref{}, we identify four representative architectural paradigms that differ in how understanding and generation pathways interact: (a) ~~, (b) models that share a language backbone but employ separate visual modules for understanding and generation, (c) models with a fully shared transformer backbone and a unified visual encoder for both modalities, and (d) ~~. From each paradigm, we select a representative open-weight model: BAGEL~\cite{}, Janus-Pro~\cite{}, Lumina-DiMOO~\cite{}, and BLIP3-o~\cite{}, respectively.
We leverage counting as the probe task since it offers a clean, quantifiable concept that both modalities must encode. The model must grasp the number of objects present regardless of whether it is answering a question or generating an image. For each model, we conduct two separate training runs using LoRA-based supervised fine-tuning. In the understanding-only condition, the model is trained on counting visual question answering data. In the generation-only condition, it is trained on counting-aware text-to-image pairs constructed from the same source images.

\subsection{Experimental setup} 
For both understanding and generation, we construct training sets based on PixMo-Points~\cite{deitke2025molmo}. For understanding, we filter PixMo-Points, originally designed for point coordinate prediction, to retain images with object counts ranging from 0 to 20 and reformulate them into counting-based visual question answering data. For generation, we use Qwen3-VL~\cite{bai2025qwen3} to produce image descriptions for the same images and append object count information to form generation prompts.

For evaluation, we use two separate protocols. Understanding is evaluated on the test split of PixMo-Count~\cite{deitke2025molmo}. For generation, we follow the counting protocol from GenEval~\cite{ghosh2023geneval} and construct prompts in the form \texttt{"A photo of [NUMBER] [OBJECT]s"} with counts ranging from 2 to 10. Generated images are evaluated using an object detector~\cite{cheng2022masked} to verify whether the number of detected objects matches the specified count. We report both accuracy and mean absolute deviation (MAD) to capture exact correctness and proximity to the ground truth.

\subsection{Results}
\subsubsection{Bi-directional transfer emerges in models with shared representations.} 
Table~\ref{tab:transfer} summarizes the results. The most consistent finding is that understanding-to-generation transfer (Und. $\rightarrow$ Gen.) is observed in all models except Janus-Pro~\cite{chen2025januspro}. BAGEL~\cite{deng2025bagel}, Lumina-DiMOO~\cite{xin2025lumina}, and BLIP3-o~\cite{chen2505blip3} all show improved generation accuracy and lower MAD when trained solely with the understanding objective, indicating that numerical knowledge acquired through understanding can generalize to generation.

The reverse direction, generation-to-understanding transfer (Gen. $\rightarrow$ Und.), is more selective. BAGEL and Lumina-DiMOO exhibit bi-directional transfer, where generation-only training also yields gains in counting understanding. Lumina-DiMOO shows the strongest effect in both directions. Notably, training Lumina-DiMOO with only the understanding objective yields even greater improvement in generation than direct generation supervision (Gen. $\rightarrow$ Gen.).

\subsubsection{()} From an architectural perspective, Lumina-DiMOO~\cite{xin2025lumina} employs a shared transformer backbone in which understanding and generation operate on the same image representation (category (c) in Figure~\ref{}). Since both objectives reference an identical representation space, any update from one objective is immediately visible to the other. This enables knowledge sharing without requiring a separate bridge or alignment module. This observation aligns with recent studies~\cite{wu2025harmonizing, jiao2025unitoken, wu2026liquid} that advocate for unified visual representations and provides empirical evidence that the degree of representation sharing is a critical factor in enabling cross-task transfer.
